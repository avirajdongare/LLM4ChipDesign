# Analysis of Synthesized 8-bit Adder Design

## 1. Architecture Identification

This is a **Ripple Carry Adder using Kill-Propagate (K-P) logic** with an inverted carry formulation.

The key insight is the unusual signal definitions:
- **P (Propagate):** `p = a ^ b` — standard propagate signal
- **K (Kill):** `k = ~(a | b)` — this is `NOR(a,b)`, true when **both inputs are 0**

The carry equation `c[i+1] = ~k[i] & (~p[i] | c[i])` can be rewritten as:
```
c[i+1] = (a[i] | b[i]) & ((a[i] & b[i]) | c[i])
       = (a[i] | b[i]) & (a[i] & b[i] | c[i])
```

This is equivalent to the standard: `c[i+1] = g[i] | (p[i] & c[i])` where `g = a & b`.

## 2. Why This Reduces Cell Count vs Traditional RCA

### Traditional RCA (per bit):
```
g[i] = a[i] & b[i]           // AND gate
p[i] = a[i] ^ b[i]           // XOR gate  
c[i+1] = g[i] | (p[i] & c[i]) // OR + AND = 2 gates
sum[i] = p[i] ^ c[i]         // XOR gate
```
**Total: ~5 gates per bit × 8 = 40 gates**

### Optimized K-P Formulation (per bit):
```
k[i] = ~(a[i] | b[i])        // NOR gate (single gate!)
p[i] = a[i] ^ b[i]           // XOR gate
c[i+1] = ~k[i] & (~p[i] | c[i]) // Implements as AOI21 or 2 gates
sum[i] = p[i] ^ c[i]         // XOR gate
```

**Key optimizations:**
1. **NOR-based kill signal** maps directly to a single cell
2. **`~p[i] | c[i]`** = `~(p[i] & ~c[i])` — this is an **AOI21/OAI21** pattern
3. The entire carry expression **maps to a single complex gate** in many libraries

**Estimated: 3-4 gates per bit × 8 ≈ 30 gates**

## 3. Trade-offs in Delay and Fanout

### Delay:
| Metric | This Design | Notes |
|--------|-------------|-------|
| Critical Path | O(n) | Still linear like RCA |
| Gate Depth | ~2n XOR + n carry stages | Similar to RCA |
| Carry Chain | Sequential through c1→c2→...→cout | No parallelism |

### Fanout Analysis:
```
Signal    Fanout
──────    ──────
p[i]      2 (carry logic + sum)
k[i]      1 (carry logic only)
c[i]      2 (next carry + sum)
```

**Low, balanced fanout** — no signal drives more than 2 gates, which is excellent for:
- Minimal buffer insertion
- Predictable timing
- Low capacitive loading

## 4. Comparison with Parallel Prefix Adders

| Adder Type | Cells (8-bit) | Delay | Wiring | Fanout |
|------------|---------------|-------|--------|--------|
| **This K-P RCA** | ~30 | O(n) | Minimal | 2 |
| **Kogge-Stone** | ~80-100 | O(log n) | Very High | O(n) |
| **Brent-Kung** | ~50-60 | O(2 log n) | Moderate | O(1) |
| **Carry Select** | ~60-70 | O(√n) | Moderate | Variable |

### Why not better than parallel prefix?
- **No parallel carry computation** — carries still ripple
- **Kogge-Stone** computes all carries in parallel with O(log n) delay
- **This design trades speed for area** — ideal for low-power, area-constrained designs

## 5. Why ABC/Yosys Achieved 30 Cells vs 40 for RCA8

### ABC's Optimization Mechanisms:

```
┌─────────────────────────────────────────────────────────┐
│  ABC Optimization Pipeline                               │
├─────────────────────────────────────────────────────────┤
│  1. AIG Representation                                   │
│     - Represents all logic as AND-INVERTER graph        │
│     - NOR(a,b) = single inverted AND                    │
│                                                         │
│  2. Structural Hashing                                   │
│     - Identifies ~(a|b) is reused in carry chain        │
│     - Shares computation across bits                    │
│                                                         │
│  3. Technology Mapping (map/amap)                       │
│     - AOI21: Y = ~((A & B) | C) maps carry perfectly   │
│     - Complex gates reduce cell count                   │
│                                                         │
│  4. Rewriting & Refactoring                             │
│     - "rewrite", "refactor", "balance" commands         │
│     - Discovers K-P form is more compact in AIG        │
└─────────────────────────────────────────────────────────┘
```
